## Начало работы

### Получение ключа

Чтобы получить Золотой API-Ключик, нужно перейти в свой профиль и внизу страницы найти кнопку
`Получить золотой API-ключик` и нажать на неё.

Если у вас ранее не было API-ключа, вы перейдёте на страницу, откуда можете его скопировать. **Сохраните ключ в надёжном
месте, он покажется только один раз! При его утере будет нужно создать новый ключ, старый при этом сбросится.**

Если у вас уже есть API-ключ, вас перенаправит на страницу с предупреждением, что новый ключ сбросит старый.

### Подготовка к отправке запроса

Полученный API-ключ необходимо добавлять в *каждый* запрос к MemeAPI в заголовке `X-Api-Key`.

#### Пример на языке Python:

```python
import requests

url = 'http://127.0.0.1:5000/api/user/1'
headers = {'X-Api-Key': 'yoursupersecretmemeapikey228'}

response = requests.get(url, headers=headers)
```

## Работа с MemeAPI

### Получение данных

На данный момент MemeAPI предоставляет возможность получать информацию о трёх видах сущностей: `user`, `post` и `meme`.
Для каждой из них определены по две функции: получение информации о сущности и фотографии, приложенной к ней.

### Формат запроса

Для каждой сущности запросы имеют общий вид:

- Общая информация о сущности: [GET] `http://127.0.0.1:5000/api/<string:entity_type>/<int:entity_id>`
- Картинка, привязанная к сущности: [GET] `http://127.0.0.1:5000/api/<string:entity_type>/<int:entity_id>/picture`

---

### Данные обо всех сущностях

С помощью MemeAPI можно получить список ID всех сущностей нужного типа. По запросу отдаются поля:

| Поле         | Тип           | Описание                      |
|:-------------|:--------------|:------------------------------|
| `type`       | string        | Тип получаемых сущностей      |
| `count`      | int           | Конечное число всех сущностей |
| `entity_ids` | array of ints | Список ID всех сущностей      |

Формат запроса: [GET] `http://127.0.0.1:5000/api/<string:entity_type>/s`

`entity_type` принимает значение одного из вида сущностей:

- `user` – пользователь
- `post` – пост
- `meme` – мем

Пример запроса: [GET] `http://127.0.0.1:5000/api/user/s`

Пример ответа:

```json
{
  "count": 5,
  "entity_ids": [1, 2, 3, 4, 5],
  "status": "ok",
  "type": "user"
}
```

Далее можно манипулировать с каждой сущностью по отдельности.

---

### Сущность типа `user`

Представляет собой пользователя, зарегистрировавшегося в MemeMatch. Отдаёт поля:

| Поле           | Тип            | Описание                                    |
|:---------------|:---------------|:--------------------------------------------|
| `id`           | int            | Уникальный ID пользователя                  |
| `username`     | string         | Уникальное имя пользователя                 |
| `nickname`     | string         | Никнейм пользователя                        |
| `about`        | string \| null | Описание пользователя                       |
| `posts`        | array of ints  | Список ID постов, созданных пользователем   |
| `created_date` | string         | Дата и время создания аккаунта пользователя |
| `status`       | string         | Статус ответа API                           |

#### Получение всех данных о пользователе

Пример запроса: [GET] `http://127.0.0.1:5000/api/user/1`

Пример ответа:

```json
{
  "about": "Я чайник!",
  "created_date": "2026-05-11 23:09:03",
  "id": 1,
  "nickname": "Чайник.",
  "posts": [],
  "status": "ok",
  "username": "teapot"
}
```

#### Получение картинки (аватарки пользователя)

Пример запроса: [GET] `http://127.0.0.1:5000/api/user/1/picture`

Формат ответа: фотография в формате `.jpg`, ограничение по размеру – 1.6 Мб.

![Пример ответа](/static/avatars/user1.jpg)

Представляет собой пост (публикацию), размещённую в MemeMatch. Отдаёт поля:

| Поле                | Тип            | Описание                                                         |
|:--------------------|:---------------|:-----------------------------------------------------------------|
| `id`                | int            | Уникальный ID поста                                              |
| `author_id`         | int            | ID автора публикации                                             |
| `meme_id`           | int \| null    | ID связанного с постом мема из таблицы мемов                     |
| `title`             | string         | Заголовок публикации                                             |
| `description`       | string \| null | Текстовое описание под карточкой мема                            |
| `likes`             | array of ints  | Список ID пользователей, которые лайкнули этот пост              |
| `matches_from_this` | array of ints  | Список ID связей-мэтчей, сгенерированных на основе этого поста   |
| `match_result`      | int \| null    | ID оригинального поста-предка, если текущий пост создан как мэтч |
| `created_date`      | string         | Дата и время выкладывания поста в ленту                          |
| `status`            | string         | Статус ответа API                                                |

#### Получение всех данных о посте

Пример запроса: [GET] `http://127.0.0.1:5000/api/post/11`

Пример ответа:

```json
{
  "author_id": 1,
  "created_date": "2026-05-16 21:16:27",
  "description": "Привет, мир!",
  "id": 11,
  "likes": [1],
  "match_result": null,
  "matches_from_this": [],
  "meme_id": 21,
  "status": "ok",
  "title": "Беды с API, а точнее с докой"
}
```

#### Получение картинки (мема, который выложен в посте)

Пример запроса: [GET] `http://127.0.0.1:5000/api/post/11/picture?type=source`

Параметры запроса:

- `type` – тип желаемого изображения:
    - `source` – исходное изображение
    - `result` – полученное в ходе встроенной в MemeMatch обработки изображение

Формат ответа: фотография в формате `.jpg`, ограничение по размеру – 3.2 Мб.
![Пример ответа](/static/uploads/sources/user1-meme21.jpg)

### Сущность типа `meme`

Представляет собой мем, созданный и выложенный в MemeMatch. Отдаёт поля:

| Поле             | Тип         | Описание                                                              |
|:-----------------|:------------|:----------------------------------------------------------------------|
| `id`             | int         | Уникальный ID мема                                                    |
| `user_id`        | int         | ID создателя мема                                                     |
| `post_id`        | int \| null | ID публикации, к которой привязан этот мем                            |
| `parent_meme_id` | int \| null | ID оригинального мема-предка, если текущий был создан как мэтч        |
| `meta`           | object      | Объект с мета-информацией о примененных фильтрах и текстах (см. ниже) |
| `status`         | string      | Статус ответа API                                                     |

#### Структура объекта `meta` из сущности `meme`

| Ключ              | Тип           | Описание                                    |
|:------------------|:--------------|:--------------------------------------------|
| `textType`        | bool          | Признак классического типа мема "С текстом" |
| `topText`         | string        | Текст верхней подписи                       |
| `bottomText`      | string        | Текст нижней подписи                        |
| `jackalType`      | bool          | Признак применения сжатия "Шакальный"       |
| `jackalDegree`    | string \| int | Степень ухудшения качества картинки         |
| `demikType`       | bool          | Признак типа мема "Демотиватор"             |
| `demikTopText`    | string        | Заголовок внутри рамки демотиватора         |
| `demikBottomText` | string        | Подзаголовок внутри рамки демотиватора      |
| `demikBorder`     | string \| int | Толщина чёрного поля демотиватора           |
| `demikOutline`    | string \| int | Толщина белой обводки вокруг картинки       |

#### Получение всех данных

Пример запроса: [GET] `http://127.0.0.1:5000/api/meme/21`

Пример ответа:

```json
{
  "id": 21,
  "meta": {
    "textType": true,
    "topText": "делал доку к api",
    "bottomText": "в итоге думал 10 минут над этим мемом для примера",
    "jackalType": true,
    "jackalDegree": "4",
    "demikType": false,
    "demikTopText": "",
    "demikBottomText": "",
    "demikBorder": "52",
    "demikOutline": "3"
  },
  "parent_meme_id": null,
  "status": "ok",
  "user_id": 1
}
```

#### Получение картинки (исходной или итоговой картинки мема)

Пример запроса: [GET] `http://127.0.0.1:5000/api/meme/21/picture?type=result`

Параметры запроса:

- `type` – тип желаемого изображения:
    - `source` – исходное изображение
    - `result` – полученное в ходе встроенной в MemeMatch обработки изображение

Формат ответа: фотография в формате `.jpg`, ограничение по размеру – 3.2 Мб.

### Взаимодействие с сервером

MemeAPI позволяет взаимодействовать с контентом непосредственно в MemeMatch.

### Лайки постов

Ставит или убирает лайк от текущего пользователя указанному посту. Отдаёт поля:

| Поле          | Тип    | Описание                                                                      |
|:--------------|:-------|:------------------------------------------------------------------------------|
| `action`      | string | Выполненное действие: `"liked"` (лайк поставлен) или `"unliked"` (лайк убран) |
| `likes_count` | int    | Актуальное общее количество лайков у поста после выполнения действия          |
| `status`      | string | Статус ответа API                                                             |

Пример запроса: [PUT] `http://127.0.0.1:5000/api/post/11/like`

Пример ответа:

```json
{
  "action": "liked",
  "likes_count": 1,
  "status": "ok"
}
```

### Удаление поста

Удаляет пост, если его автор – текущий пользователь API.

Пример запроса: [DELETE] `http://127.0.0.1:5000/api/post/12`

Пример ответа:

```json
{
  "status": "ok"
}
```

---

### Общий формат ответа

В каждом JSON-ответе MemeAPI гарантированно есть поле `status`. Оно принимает значения:

- `ok` – при успешной отработке MemeAPI
- `error` – при ошибке в отработке MemeAPI; в этом случае в ответе указывается поле `message` с человекочитаемым
  объяснением ошибки

#### Пример ошибочного запроса с указанием несуществующего ID поста

Запрос: [GET] `http://127.0.0.1:5000/api/post/123`

Ответ:

```json
{
  "message": "The post №123 is a lie",
  "status": "error"
}
```

### Ограничения

MemeAPI допускает до тридцати запросов с одного ключа, лимит сбрасывается каждые 10 секунд.
